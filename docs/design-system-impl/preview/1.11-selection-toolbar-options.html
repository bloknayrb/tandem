Selection toolbar — affordance options

Conflict #5 is locked at the principle level: the production audience-first popup stays primary; the bundle's formatting controls become a secondary affordance, collapsed by default, expandable. What's open is the shape of that affordance. Below: what ships today, the bundle reference (not adopted wholesale — it regresses ADR-027), then three candidate shapes. All preserve popup-* testids.

Pills are interactive — click expanders, Annotate, swatches. Toggle the theme to sanity-check contrast.

Production today

Ships now · primary

Audience-first. Single row: B / I, highlight swatches, then Annotate (outlined in the user-blue). Clicking Annotate swaps the pill to the note card — Enter = Comment (to Claude), or "Note to self" (private). Strike / inline-code / link were intentionally dropped from the popup; they live on the persistent FormattingBar.

Bundle A8 reference

Reference only · not adopted

Formatting-first two-row pill with a coral comment icon on the right. No audience awareness — adopting it wholesale regresses ADR-027's "Note vs Send to Claude" model. We mine its formatting controls and coral comment treatment, not its information architecture.

A8 — two-row formatting + coral comment

Option A — chevron reveals a second row

Recommended

Audience row is unchanged and primary. A chevron at the right edge expands a second row below with the extra formatting controls (S, code, link, lists, quote, heading). Default = collapsed = audience-first intact.

Click the ⌄ chevron

Option B — "Aa" toggle swaps the row

Alternative

A single Aa button on the audience row. Clicking it replaces the row contents with the full formatting set plus a back arrow; the pill never grows taller. Tradeoff: formatting and audience are never visible at the same time.

Click Aa, then ←

Option C — chevron grows the row inline

Alternative

Same chevron as A, but the extra controls slide in on the same row rather than dropping a second row. Keeps a single-row silhouette; tradeoff: the pill gets wide and may need edge-clamping near the viewport border.

Click the ⌄ chevron

  • Coral comment treatment: keep production's outlined-blue Annotate, or adopt A8's coral comment icon? (Conflict #6 — coral would need a WCAG check.)
