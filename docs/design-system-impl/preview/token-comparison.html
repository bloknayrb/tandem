Dark mode canvas — Prod wins

The divergence: bundle uses hue 270° / lightness 0.18 (pre-v7 baseline); production uses hue 280° / lightness 0.22 (calm-v7 canon shipped via #776 and #621). Side-by-side, the bundle's dark canvas reads as colder and deeper; production reads warmer and slightly lifted. The fg color also differs in hue (bundle 80° warm vs production 280° cool) — bundle's warm fg can read as off-balance against the cool dark surface.

What this means for sub-PRs

Production stays on the calm-v7 dark canvas. Bundle colors_and_type.css remains the source of truth for everything ELSE — type scale, spacing, radius, shadows, semantic type roles, utility classes, light-mode tokens, dark-mode authorship + status + suggestion + highlights. Sub-PRs touching token usage should reference the right-hand "Bundle · ..." column for everything except the dark canvas, fg-pair, and the protected tokens, which all stay on production values.
