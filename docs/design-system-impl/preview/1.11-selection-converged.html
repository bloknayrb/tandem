Sub-PR 1.11 · optional persistent bar + full selection popup (scratch — not committed)

One full selection popup, an optional bar

The selection popup is always the full surface — a format pill (inline marks + block type) stacked over an annotate pill (highlight + Annotate). The persistent formatting bar becomes optional: a swap control hides or shows it. undo / redo and the authorship toggle move to the title bar so they're available whether or not the bar is shown.

Click Show / Hide bar in the live demo to swap; click Annotate to open the note popover. Switch themes to check contrast.

Theme

The model

Three surfaces. The title bar cluster (undo / redo / authorship) is always present. The persistent formatting bar is optional — formatting only, with a hide control. The selection popup is always the full stacked pills. The two modes differ only in whether the persistent bar is present; the popup is identical either way.

Live — swap + Annotate

The swap control shows / hides the persistent bar. The selection popup stays full in both modes. Clicking Annotate swaps the pills for a note popover anchored at the same spot — it flips above/below the selection so the highlighted selection stays visible; no quoted-text echo needed. Two explicit exits: Note to self (private; ⏎) and Send to Claude (outbound; ⌘⏎, primary). Esc or click-outside cancels.

Behavior I'll build to

  • The selection popup is always full: format pill (B / I / S / code / link · heading / lists / quote / code block) over an annotate pill (highlight swatches + Annotate).
  • New setting formattingBarVisible: boolean (default true = today's behavior). Persisted in tandem:settings; Settings is the canonical home, the in-surface swap control is the convenience.
  • undo / redo + authorship toggle relocate to the title bar (always visible). undo / redo also get an editor right-click menu entry. The persistent bar becomes formatting-only.
  • Annotate swaps the pills for the anchored note popover; Esc / click-outside cancels.
  • Note to self → private (audience: "private", ⏎). Send to Claude → outbound (audience: "outbound", ⌘⏎, primary/filled).
  • Suggestions stay Claude-only — the popover never offers a "suggestion" exit (locked rule).
  • popup-* testids preserved; new surfaces get their own (e.g. formatbar-visibility-toggle, titlebar-undo, titlebar-redo, annotate-popover, annotate-note-submit, annotate-send-submit).
A 1.0 build, not just a re-skin. The bar isn't deleted (no Conflict #2 override) — it's made hideable and reduced to formatting-only. But moving undo/redo + authorship to the title bar and growing the popup to full formatting is real behavior: a settings key + migration, a conditional FormattingBar render, a title-bar cluster, a right-click menu, and reverting 1.2's formatbar-authorship-toggle placement. Likely a 2-PR sequence (popup + swap/settings; then control relocation + bar trim). Sequenced after the remaining Phase-1 visual sub-PRs, since it's the most invasive one.
