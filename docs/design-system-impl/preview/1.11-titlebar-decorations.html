Title bar: a Decorations control

Authorship stops being a lone toggle and becomes one row in a Decorations control — a split button: the eye mutes / restores all decorations at once (clean reading view ⇄ restore), the caret opens the per-type options. Four rows: Authorship colors, Comments, Highlights, Notes. undo / redo stay where they are (formatting bar + Ctrl+Z/Y); a right-click menu is the future home, not 1.11.

Decorations feature — plan

  • Four top-level toggles: Authorship colors, Comments, Highlights, Notes. showAuthorship already exists; the other three are a per-type split of today's single showAnnotationDecorations (#596). Claude's focus is dropped — it only appears in Tandem mode, so the Solo/Tandem toggle already covers it. Annotation patterns (a11y) stays in Settings, not the top-level menu.
  • Per-type split. Today showAnnotationDecorations shows/hides all annotation marks. annotation.ts's buildDecorations instead filters by annotation type against three flags — showComments (type "comment", incl. suggested-text comments), showHighlights ("highlight"), showNotes ("note"). Display-only — does not touch ADR-027 (notes are still never read by Claude; this only hides the user's own marks in their view).
  • Consolidation, not invention. Today authorship lives on the formatbar button and the annotation toggle in Appearance settings. The feature gathers the display toggles into one Decorations split button + keeps a mirrored group in Settings → Appearance (canonical home; a11y patterns stay there too).
  • Wiring reuses the established pattern — each toggle is an $effect that dispatches setMeta(pluginKey, {type, visible}) to its ProseMirror plugin (exactly how authorship + #596 already work in App.svelte).
  • Settings model: replace showAnnotationDecorations with showComments / showHighlights / showNotes (default true; the migration maps the old single flag onto all three) + a decorationsMuted master flag + a useTandemSettings bump. Client-only — no src/server/ changes.
  • Master = mute / restore. The eye half: clicking when anything is on remembers the current set and hides all decorations (clean reading view); clicking when muted restores exactly that set. Implemented as a decorationsMuted overlay flag that suppresses rendering without clobbering the individual prefs; editing a row in the dropdown auto-unmutes.
How this relates to 1.11. 1.11 needed an always-visible home for the authorship toggle once the formatting bar became hideable. Folding authorship into the Decorations control solves that and answers your "make all decorations toggleable" ask in one move. Sequencing: 1.11 = full popup + optional bar + swap/settings (the bar just loses its authorship toggle; undo/redo stay put). 1.13 · Decorations control = the title-bar split button + the per-type annotation/notes filter split. undo/redo are untouched in both; a right-click menu is a separate later follow-up.
