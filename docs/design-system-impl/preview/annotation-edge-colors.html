The discriminator is background tint (not a left-border edge color); the header carries the author dot, name, type badge, and timestamp; the actions row is type-specific. This is the load-bearing visual rule for Phase 3.3 — protects ADR-027 + Conflict #8.

Anti-patterns the taxonomy forbids:
  • No left-border edge color. The card border is the standard 1px var(--tandem-border) all the way around. Background tint is the primary signal.
  • No if (annotation.author === "claude") branch in any card render path — the production card variant (NoteCard / CommentCard / SuggestionCard / ImportedCard / HighlightCard) is the predicate.
  • Suggestions are Claude-authored only. Users author notes, comments, and highlights — never suggestions. tandem_createAnnotation must reject a user-authored annotation with suggestedText set, and the UI must never render one.
  • Imported cards have no author dot — the italic byline replaces it.
  • Notes never show a type badge — they're audience-private; the absence of a badge is the visual signal.
  • Highlights are a separate surface entirely — they render as inline editor marks, no card chrome, color from HIGHLIGHT_COLOR_VARS.
  • Do not change the three protected tokens (--tandem-author-user, --tandem-author-claude, --tandem-suggestion) without re-running token-protection.test.ts.
