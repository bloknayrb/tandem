### Pre-stage check command

Run before starting each Stage to detect concurrent Session B work intersecting Stage C/D.
The script fails LOUDLY (not silently with empty output) on `gh` auth failure, rate limit,
or network down — a silent empty result on auth failure was the bug Session A reviewer
flagged in PR #909's hardening pass.

```sh
set -euo pipefail
gh auth status >/dev/null 2>&1 || { echo "ERROR: gh not authenticated; check did not run."; exit 1; }
git fetch origin
prs=$(gh pr list --base feat/design-system-impl --state open --json number --jq '.[].number')
if [ -z "$prs" ]; then
  echo "VERIFIED: no open PRs against feat/design-system-impl."
  exit 0
fi
intersections=$(echo "$prs" | xargs -I{} gh pr view {} --json files \
  --jq '.files[].path | select(test("MarginColumn|useMarginPositions|AnnotationCard|marginCollision|derived-spec|conflicts-resolved"))')
if [ -z "$intersections" ]; then
  echo "VERIFIED: $(echo "$prs" | wc -l) open PR(s), none touch Stage C/D files."
else
  echo "INTERSECTION: concurrent PR(s) touch Stage C/D files — coordinate before continuing:"
  echo "$intersections"
fi
```

`VERIFIED: ...` lines distinguish "ran clean, no intersection" from "didn't run." An
`INTERSECTION: ...` block means a concurrent PR touches Stage C/D files — coordinate
before continuing.

**One Stage-A blocker** identified and inline-resolved in §5:

- **C7 (HIGH) — docx/non-docx fork on `marginLayerEl`.** The naïve "grid stage IS
  `marginLayerEl`" wording would have re-bound the layer on every docx tab switch
  (silently violating the no-re-bind invariant the section claimed to preserve "by
  construction"). **Resolution folded into §5:** keep `bind:this={marginLayerEl}` on
  the **outer wrapper** that spans both `{#if format === "docx"}` arms (today's
  `App.svelte` L1576 div); make the grid container an *inner child* of the non-docx
  arm, **without** its own `bind:this`. Invariant 3's border-box-top rule now applies
  to the outer wrapper (the bound element), which frees the grid to carry padding/
  border. This is the only change that materially affects Stage A's structure.

**Six hardenings** folded into their relevant sections (all MEDIUM, do-in-stage):

- **C8** — JSDOM mount-time assertion that `marginLayerEl` has zero `padding-top` /
  `border-top` (tripwire against future drift; Invariant 3 is comment-hope without it).
- **C9** — SVG leader horizontal `getBoundingClientRect()` reads must share a rAF tick
  with `useMarginPositions`'s `layerTop` recompute, else leaders chase bubbles by one
  frame during track-width transitions.
- **C10** — Mode `$effect` requires an explicit `if (next === lastMode) return;` last-
  value guard (per `feedback_svelte_effect_depth_guard`). The narrowSticky analogy
  isn't enough — needs the literal short-circuit.
- **C11** — Reduced-motion guard must be **dual-mechanism**: `@media (prefers-reduced-
  motion: no-preference)` AND `body:not(.tandem-reduce-motion)`. Single mechanism
  leaks motion to users who disabled it via the other surface.
- **C12** — Schema v12 forward-compat: `useTandemSettings.ts` must derive `knownKeys`
  at runtime from the schema (not hardcoded), per
  `feedback_forward_compat_runtime_knownkeys`. Verify before landing v12.
- **C13** — Stage C must spec the 22px stub-pill's minimum metadata surface (6px
  author dot retained, status pip via `cardTint` border-color) so 1.5's review-target
  affordance survives. Explicit guardrail: clamp/stub are CSS density modifiers — **
  `cardTint` keeps exactly 6 branches**, no 7th "stub" branch.
- **C14** — `createEditorStageModel` must be invoked inside `<script>` (effect root)
