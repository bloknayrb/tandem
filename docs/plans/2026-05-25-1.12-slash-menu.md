# Sub-PR 1.12 — Slash Menu visual port

## Context

Umbrella `feat/design-system-impl`. Sub-PRs 1.1–1.10 merged; 1.11 (SelectionToolbar) is blocked on Bryan's design direction, so 1.12 proceeds in parallel. **1.12 is NOT one of the nine flagged conflicts** in `conflicts-resolved.md` — it's a "clean port" — but it carries the same shape as 1.9/1.10: the bundle mockup contains structural elements that don't fit production's interaction model. Per `feedback_bundle_aesthetic_production_information`: adopt bundle aesthetics for decoration, preserve production where it carries information.

**Bundle source:** `docs/design-system-impl/bundle/extracted/B3 - Slash Menu.html`.
**Production surface:** `src/client/editor/slash-menu/` (`extension.ts` renders vanilla DOM; `commands.ts` is the registry; `suppression.ts` is the D10 rule) + the `.tandem-slash-menu*` CSS in `index.html` (L276–326).

## What production is today

A Tiptap ProseMirror plugin. Typing `/query` inline in the document triggers it; `findSlashCommandMatch` reads `textBeforeCursor`, `filterSlashCommands` filters the 8-command registry. The plugin's `view()` appends a `<div data-testid="slash-command-menu">` to `document.body`, renders a `role="listbox"` of `role="option"` `<button>`s (label text only), plus a keyboard-hint footer (`.tandem-slash-menu__hint`: ↑↓ navigate / ↵ insert / esc close). Selection: ArrowUp/Down wrap, Enter executes, Escape closes, mouseenter sets selected, mousedown executes. Empty results are filtered upstream (`resolveActiveSlashCommand` returns null when `items.length === 0`), so **the menu closes on zero matches — an empty state never renders.**

## What the bundle adds (B3)

1. Per-row **icon badge** (26×26, `--tandem-r-3`, `--tandem-surface-sunk` bg, 1px border; tints to accent on selected row).
2. Per-row **shortcut-hint kbd** on the right (`p`, `h1`, `ul`, …) — `.o1-hint`, mono, bordered.
3. **Card shape**: 256px wide, 16px radius, heavier popover shadow.
4. A separate **filter-input row** (search icon + `Filter blocks…` input).
5. An **empty state** (`No matching blocks`).
6. A 9th command **Paragraph**, and different labels (`Blockquote`, `Divider`).
7. A `popIn` entry animation.

## Scope decision (the load-bearing call)

**This is a visual-only port. Port the decorative layer; drop the mockup artifacts; freeze the registry.**

| Bundle element | Decision | Why |
|---|---|---|
| Per-row icon badge | **Port** | Core visual identity of B3. |
| Per-row shortcut-hint kbd | **Port** | Decorative; derive label from existing keyword (no registry change). |
| Card width (~256px) + radius (16px ≈ `--tandem-r-5`) | **Port** | Visual. |
| Selected-row: drop border, keep fill + accent text | **Port** | Matches bundle; avoid layout shift (see below). |
| Filter-input row | **DROP** | Mockup artifact. Production filters via inline `/`-typing — the query is already visible at the cursor. A second, unwired input would duplicate/confuse. |
| Empty state | **DROP** | Production closes the menu on zero matches; an empty state is unreachable in this interaction model. (No behavior change — keep close-on-empty.) |
| 9th "Paragraph" command | **DROP** | Registry change. Out of scope for a visual port; the registry is frozen (consistent with 1.6 CommandPalette). |
| Relabel Quote→Blockquote, Horizontal rule→Divider | **DROP** | Registry/content change; E2E asserts `name: "Quote"` / `"Horizontal rule"`. Keep production labels. |
| `popIn` keyframes | **DROP** | Conflict #9 — no new animation choreography in Phases 1–3 (deferred to #798). Keep whatever exists (nothing animates today). |

**Result:** 8 commands, existing labels, same trigger/filter/keyboard behavior. Only the rendered markup of each row + the card chrome change. No `src/server/` change.

## Concrete edits

### 1. `src/client/editor/slash-menu/commands.ts`

Add two display fields to `SlashCommandItem`:

```ts
export type SlashCommandIcon =
  | { kind: "glyph"; glyph: string }      // text glyph, e.g. "H¹", "—"
  | { kind: "svg"; paths: string[] };     // SVG path `d` strings (createElementNS)

export interface SlashCommandItem {
  id: SlashCommandId;
  label: string;
  keywords: string[];
  hint: string;        // short alias shown on the right (e.g. "h1", "ul")
  icon: SlashCommandIcon;
  run: (editor: TiptapEditor) => void;
}
```

Per-command `hint` / `icon` (hints come straight from existing keywords — no new vocabulary):

| id | hint | icon |
|---|---|---|
| heading-1 | `h1` | glyph `H¹` |
| heading-2 | `h2` | glyph `H²` |
| heading-3 | `h3` | glyph `H³` |
| bullet-list | `ul` | svg (3 dots + 3 lines, from B3) |
| numbered-list | `ol` | svg (1/2 marks + lines, from B3) |
| quote | `q` | svg (quote marks, from B3) |
| code-block | `code` | svg (chevrons `</>`, from B3) |
| horizontal-rule | `hr` | glyph `—` |

SVG path arrays lifted verbatim from B3's inline `<svg>` markup (same viewBox `0 0 16 16`). This mirrors the `SEVERITY_GLYPHS` path-array precedent from 1.10 (`activityCenter.ts`) — injection-free, lint-clean, no `innerHTML`.

### 2. `src/client/editor/slash-menu/extension.ts`

In `renderSlashCommandMenu`, for each option `<button>`:
- Keep `role="option"`, `id`, `aria-selected`, `data-commandId`, mouseenter/mousedown handlers **unchanged**.
- **Set `button.setAttribute("aria-label", command.label)`** and build children as decorative nodes marked `aria-hidden`, so the button's accessible name stays exactly the label.
  - **a11y note (correct causality):** Playwright's `getByRole(role,{name})` is substring + case-insensitive, so the role-based E2E selectors would pass even with a polluted name. This is **hygiene, not a test-unblock** — keeping the accessible name == label avoids future ambiguity (e.g. a hint of `h2` matching multiple options) and is the right ARIA shape. (Per advisor.)
- Children: `<span class="tandem-slash-menu__icon" aria-hidden="true">` (badge: glyph via `textContent`, or SVG via `createElementNS`), then `<span class="tandem-slash-menu__label">` (label text — replaces the bare `textContent`), then `<span class="tandem-slash-menu__shortcut" aria-hidden="true">` (the hint). **Do NOT add `tabindex` anywhere** — options stay non-tab-stops; Tiptap keeps focus in the editor view (advisor verification point).
- Add an SVG-icon builder using `document.createElementNS("http://www.w3.org/2000/svg", …)` for `kind:"svg"` (svg + path elements, set `viewBox`, `fill=none`, `stroke=currentColor`, etc. to match B3), `textContent` for `kind:"glyph"`.
- `buildHintRow()` (footer) is **unchanged**.
- The `listbox`/footer structure, positioning (`coordsAtPos`), and `executeSlashCommand` are **unchanged**.

### 3. `index.html` CSS (L276–326)

- `.tandem-slash-menu`: keep inherited `tandem-floating-pill` bg/border/shadow (**no parallel shadow scale** — keep `--c7-pill-shadow`). Change `border-radius: var(--tandem-r-4)` → `var(--tandem-r-5)` (16px ≈ bundle). Set `min-width: 256px`. Keep `padding: var(--tandem-space-1)` for the list area.

- `.tandem-slash-menu__item:hover`: `background: var(--tandem-surface-sunk)` (matches B3 `.o1-row:hover`).

- `.tandem-slash-menu__item[aria-selected="true"] .tandem-slash-menu__icon`: accent tint (`background: color-mix(in srgb, var(--tandem-accent) 14%, var(--tandem-surface)); border-color: var(--tandem-accent-border); color: var(--tandem-accent);`) per B3 `.o1-row.on .blk-icon`.
- **New `.tandem-slash-menu__icon`** (badge): `flex-shrink:0; width:26px; height:26px; border-radius: var(--tandem-r-3); background: var(--tandem-surface-sunk); border: 1px solid var(--tandem-border); display:grid; place-items:center; color: var(--tandem-fg-subtle); font-weight:700; font-size: var(--tandem-text-2xs); line-height:1;` plus `.tandem-slash-menu__icon svg { width:14px; height:14px; display:block; }`.

- `.tandem-slash-menu__hint` (footer) + its `kbd`: **unchanged** — already matches B3's footer.
- No raw hex / non-neutral rgba introduced; everything resolves to existing tokens. `npm run check:tokens` passes by construction.
