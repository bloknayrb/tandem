Tandem · interface redesign · 2026

A document in conversation
with the writer beside you.

The current Tandem chrome is workmanlike but loud — buttons live far from where the writing happens, and Claude's presence reads as a side panel rather than a co-author. This redesign brings tools to the cursor, settles the chrome down, and lets the prose breathe. Every artboard below is an actual desktop window state.

01
Audience-first annotation. The selection popup asks who is this for — Note (private) or Comment (to Claude). Two buttons, one decision. Highlights stay quick with four color swatches (yellow / green / blue / pink).
02
Authorship gutter. A 2px thread left of every paragraph shows who last touched it — blue for you, coral for Claude. Pulses softly when Claude is active. Togglable in Accessibility settings.
03
Quieter chrome. Thin top bar with brand, mode toggle (Solo / Tandem), and panel controls. Tabs are a 30px secondary row. Status bar carries inline name editor and Claude presence dot.
04
3-type data model. highlight / comment / flag — visual variants (suggestion, note, imported) derived via suggestedText and directedAt discriminator fields. Settings: stdio + HTTP only, all About values dynamic.
